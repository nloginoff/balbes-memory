# Heartbeat Checklist

## What to check each run

1. **Pending tasks** — Review MEMORY.md for any tasks or follow-ups that the user mentioned but haven't been completed yet. If something looks urgent or overdue, mention it.

2. **Friendly check-in** — If it's been a while since the user was active and it's daytime, send a brief, natural "anything you need?" type of message. Don't do this every run — only if it feels appropriate (e.g., once per day at most). Vary the wording.

3. **System reminders** — If MEMORY.md contains scheduled reminders (e.g., "remind me on Tuesday to..."), check if any are due today or soon.

## Response rules

- If nothing needs attention: reply exactly `HEARTBEAT_OK`
- If you have something to say: write a concise, friendly message WITHOUT `HEARTBEAT_OK`
- Keep messages short (1-3 sentences max)
- Do NOT repeat yourself across heartbeat runs — track what you've already mentioned

## Style
Casual and warm. Like a helpful colleague who checks in occasionally, not a nagging alert system.
